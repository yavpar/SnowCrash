## ❓ Preguntas Pendientes (Que debemos responder)

Antes de comenzar a escribir la teoría y ejercicios, necesito saber:

### **1. ¿Cuánto tiempo tienes disponible?**

- [ ] **Opción A**: Este es un proyecto de **largo plazo** (3-6 meses)
  - Podemos ser muy detallados y crear mucho contenido
  
- [ ] **Opción B**: Necesito aprenderlo en **plazo corto** (3-4 semanas)
  - Priorizamos lo esencial, menos contenido pero igual de efectivo
  
- [ ] **Opción C**: Depende - **voy a mi ritmo**
  - Creamos contenido por bloques, completas cuando puedas

---

### **2. ¿Teoría + Práctica juntas O separadas?**

- [ ] **Opción A**: **Mezcladas** (recomendado)
  ```
  Lee lección 1 (10 min)
      ↓
  Haz ejercicio 1 (20 min)
      ↓
  Lee lección 2 (10 min)
      ↓
  Haz ejercicio 2 (20 min)
  ```
  
- [ ] **Opción B**: **Separadas** (aprende todo primero)
  ```
  Lee lección 1,2,3,4,5,6,7,8 (completa)
      ↓
  Haz ejercicio 1,2,3,...,13 (completa)
  ```

---

### **3. ¿Soluciones completas O solo pistas?**

- [ ] **Opción A**: **Soluciones completas**
  - Cada ejercicio tiene `.sh` o `.c` con la solución funcionando
  - Incluye explicación línea por línea
  - Mejor para: aprender rápido, verificar si está bien
  
- [ ] **Opción B**: **Solo pistas/hints**
  - Archivo `notas.md` con guías pero no la solución completa
  - Mejor para: desafío mayor, razonamiento independiente
  
- [ ] **Opción C**: **Ambas**
  - Pistas primero
  - Solución al final
  - Mejor para: máximo aprendizaje

---

### **4. ¿Necesitas guardar como PDF o solo GitHub?**

- [ ] **GitHub** (actual, es suficiente)
  - Markdown se ve bien en GitHub automáticamente
  - Fácil de actualizar
  - Se puede compartir el link
  
- [ ] **PDF descargable**
  - Útil para lectura offline
  - Más formal
  - Requiere trabajo extra
  
- [ ] **Ambos**
  - GitHub para trabajo actual
  - PDF generado automáticamente para descargar

## 🗂️ Formato Recomendado para el Contenido

Cada elemento tendrá esta estructura:

### **Para Teoría** (`leccion_XX.md`):
```markdown
# Lección XX: [Nombre]

## Objetivos
- Qué aprenderás
- Por qué es importante

## Conceptos Fundamentales
### Concepto 1
[Explicación explícita]

[Ejemplo concreto]

### Concepto 2
[Similar...]

## Resumen
[Quick recap]

## Siguiente
Lee [Lección XX+1] y haz [Ejercicio]

## 🔮 Hint Futuro (si aplica)
[Cómo esto conecta con proyectos posteriores]
```

### **Para Ejercicios** (`ejercicio_XX.md`):
```markdown
# Ejercicio XX: [Nombre]

## Objetivos de Aprendizaje
- Qué aprenderás

## Instrucciones
1. Paso 1
2. Paso 2
3. ...

## 💡 Pistas (sin revelar)
Si estás atrapado:
- Hint 1
- Hint 2

## Solución
[Código completo]

[Explicación línea por línea]

## Variaciones
Intenta:
- Modificación 1
- Modificación 2
```

### **Para Archivos de Código** (`solucion_XX.sh` / `solucion_XX.c`):
```bash
#!/bin/bash
# Solución del Ejercicio XX
# Descripción breve de qué hace

# Paso 1: Explicar
echo "Haciendo XYZ"

# Paso 2: Código
comando_aqui

# ... etc
```
